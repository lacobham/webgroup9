The Problem

Before we could create a solution to any issue, our team needed to identify a problem that could be solved using the web. Below is the breakdown of our problem statement, which covers the ideal, the reality, the consequences, our proposal, and the software we will use in our solution.

Ideal

A user friendly web application for the public to easily report and view data on live/dead turtle sightings near or on roads in Ontario.

Reality

There is no public map that displays this data in an interactive and easy to navigate way.

Consequences

High-risk areas with turtle populations cannot be identified, causing populations to decrease without positive human interference/mitigation.

Proposal

An user friendly web application that displays the data inputted by the public that contains pop-ups/highlights areas of high risk turtle crossings. The web application will allow conservation authorities access to public science information to help in decison making and locate high risk areas.

Software

ArcGIS Survey 123, ArcGIS Dashboard, ArcGIS StoryMaps.
