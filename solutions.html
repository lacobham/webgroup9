We learned how to utilize layer views in our Dashboard, by filtering fields we wanted to keep anonymous and creating a study area to limit the data shown on the web map. We hid fields like name and email, as well as the automatic fields added by Esri. As well, we drew a polygon around Ontario so any turtles sighted outside could be reported but would not be shown.

You can see that the layer view worked because when querying the layer in the REST Endpoint, it shows 10 results, or 10 turtle sightings.

Enhancements and Advanced Formatting

The main thing that was learned in depth and explored more when completing the dashboard was how to further enhance certain traits of specific elements, such as the indicator and the list elements. Both of these have the options to add advanced formatting to them, as well as add in-line CSS to some of the text and displays – such as adding padding around the titles of most of the elements to ensure equal spacing.

An example of additional HTML/CSS formatting can be seen in the title of the list element on the dashboard, titled ‘Turtles in area’. A screenshot can be seen here:

The CSS added to the source option in the title formatting allows it to have a darker green background than the rest of the table, a larger font size than the normal heading sizes, and added padding on the bottom to make the title centered vertically in the green border as well.

As mentioned before, along with adding CSS to certain text, Dashboards also gives you the option to do advanced formatting to further enhance your elements and data. A screenshot can be seen below of the advanced formatting window of the same list element:

When writing advanced formatting, the code is in the Arcade language. This script is creating a new variable to show in the list called ‘timeFormatted’. This script is giving the list a way to show the amount of time between when the turtle was seen and the current time. Limitations

Although these methods give a nice way to look at the data, there are some limitations of dashboards. These limitations are noted below:

  • - When filling out the source CSS/HTML and switching from the source screen (where the CSS is inputted) to the format screen, if anything is incorrectly typed in, the format will change to include the entire code (which will then display on the format) and replace certain symbols with their UTF-8 entities (‘<’ will become ‘<’). If this occurs, there is no way to undo it to get the old working code back, meaning it will have to be retyped again.
  • - Without properly formatting the title of elements to be inside a ‘div’ or something similar, if the dashboard gets downsized enough, the titles inside elements will be cut out or covered over by the rest of the information in the element, making them unreadable without expanding them, which covers the rest of the information on the dashboard.

Custom Icons on Dashboards

A cool tool we learned was creating custom icons inside of Dashboard indicators using free SVG files. To create these icons all you have to do is go into the "Indicator" tab and click on the "Add icon", inside you will have the option to choose a custom icon. This is where you insert the SVG code that connects to your icon, to get this just download the SVG file and open its page source to get the code. Stacking Element on Dashboard

There is a visualization trick in Dashboard that allow you to stack elements together like graphs, indicators, and gauges into one element. Each element added gets its own tab that the user can rename. This allows the user to add more information onto the Dashboard since it reduces space. This can be done by adding new elements on top of each other and holding "Shift", this will stack them together within the same space. Below is an example of the workflow within Dashbaord.
